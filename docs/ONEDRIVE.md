# OneDrive Integration

How the Hub integrates with Microsoft OneDrive for document storage.

---

## Purpose

All formal documents generated by the Hub are saved to OneDrive:
- Supplier invoices (attached during receiving)
- Purchase orders
- Reports

This provides a permanent, organised filing system outside the app.

---

## Authentication

Uses Microsoft OAuth2 (Authorization Code flow) via Azure App Registration.

**Required env vars:**
```
ONEDRIVE_CLIENT_ID=
ONEDRIVE_CLIENT_SECRET=
ONEDRIVE_TENANT_ID=
ONEDRIVE_REDIRECT_URI=
```

**Setup flow:**
1. Go to Settings → OneDrive → Authorise
2. Redirects to Microsoft login
3. On approval, refresh token is stored in Redis
4. Token is refreshed automatically on each use

**Routes:**
- `GET /api/onedrive/auth` — initiates OAuth flow
- `GET /api/onedrive/callback` — handles OAuth callback, stores token

## Folder Structure

Created automatically via `POST /api/onedrive/setup-folders`.

```
Paynter Bar/
├── Invoices/
│   ├── Dan Murphys/
│   ├── Coles Woolies/
│   └── ACW/
├── Purchase Orders/
└── Reports/
```

---

## Document Filing

### Invoices
Filed when a delivery is confirmed in the Receive modal.

- Route: `POST /api/onedrive/save-invoice`
- Path: `Paynter Bar/Invoices/{Supplier}/{PO-Ref}-Invoice.{ext}`
- Filename uses the PO Reference from the receive modal

### Purchase Orders
Filed when a PO is confirmed and sent.

- Route: `POST /api/onedrive/save-po`
- Path: `Paynter Bar/Purchase Orders/{PO-Ref}.pdf`

### Reports
Filed for SOH reports and other generated reports.

- Route: `POST /api/onedrive/save-report`
- Path: `Paynter Bar/Reports/{filename}`

## Lib Functions (`lib/onedrive.js`)

```javascript
getAccessToken()           // Returns valid access token (refreshes if needed)
saveFile(path, buffer)     // Save file buffer to OneDrive path
createFolder(path)         // Create folder if not exists
setupFolders()             // Create full folder structure
```
